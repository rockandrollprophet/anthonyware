# 🔌 Anthonyware USB Installer — Complete Setup Guide

## Overview

This guide walks you through creating a **production-ready USB installer** for Anthonyware OS with:

- ✅ **Automatic autorun** on boot
- ✅ **Safety lock** to prevent accidental installs
- ✅ **Integrity checking** to verify repo completeness
- ✅ **Self-update** functionality for keeping repo fresh
- ✅ **Ventoy support** for custom boot menu
- ✅ **Zero manual configuration** after first setup

## Setup Instructions

### Step 1: Prepare the USB with Ventoy

Ventoy is the cleanest way to create a multiboot USB. It lets you:
- Boot multiple ISOs without rewriting the USB
- Keep persistent storage for your repo
- Add custom boot configurations

**Install Ventoy**:

```bash
# Download Ventoy from https://www.ventoy.net
# On your target USB (will be erased):
sudo bash Ventoy2Disk.sh -i /dev/sdX

# (Replace sdX with your USB device)
```

This creates:
- A boot partition with Ventoy
- A data partition for files

### Step 2: Copy Files to USB

Mount the USB data partition and copy your files:

```bash
# Mount USB (typically /dev/sdX2)
mkdir -p /mnt/usb
sudo mount /dev/sdX2 /mnt/usb

# Copy Arch ISO
cp archlinux-*.iso /mnt/usb/

# Copy your repository
cp -r ~/anthonyware /mnt/usb/

# Unmount
sudo umount /mnt/usb
```

Result:
```
/mnt/usb/
├── archlinux-2024.01.01-x86_64.iso
└── anthonyware/               (full repo)
```

### Step 3: Boot Arch ISO from USB

```bash
# Insert USB
# Boot from USB (F12, ESC, or DEL during startup)
# Select "Arch Linux" from Ventoy menu

# System boots into Arch live environment
```

### Step 4: Install Autorun Service (One-Time Setup)

Once in the Arch live environment:

```bash
# Mount USB data partition
mkdir -p /mnt/usb
mount /dev/sdX2 /mnt/usb

# Navigate to repo
cd /mnt/usb/anthonyware

# Install autorun service into live environment
sudo bash usb/install-autorun-into-live.sh

# This outputs:
# [autorun-setup] ✓ Service installed
# [autorun-setup] ✓ Binary installed
# [autorun-setup] ✓ Service enabled
```

### Step 5: Arm the USB (Enable Autorun)

To make the installer run automatically on next boot:

```bash
# Still in the Arch live environment
cd /mnt/usb/anthonyware

# Create safety lock marker
touch USB_AUTORUN_ENABLED

# Verify it's there
ls -la USB_AUTORUN_ENABLED
```

### Step 6: Reboot

```bash
sudo reboot
```

The system will:
1. Boot Arch ISO
2. Load live environment
3. Autorun service starts
4. Scans USB for `anthonyware/` directory
5. Checks for `USB_AUTORUN_ENABLED` marker
6. If present → runs `run-from-usb.sh`
7. Starts full installation automatically

## Usage Workflows

### Workflow 1: Standard Installation (USB with Autorun)

```bash
# 1. Boot USB
# 2. System boots Arch ISO
# 3. (wait ~30 seconds)
# 4. Autorun detects USB and starts installer
# 5. Answer prompts for disk, hostname, user, etc.
# 6. Installation proceeds automatically
# 7. System reboots into fully configured Anthonyware OS
```

**Time**: 5 minutes of setup + 45-90 minutes installation

### Workflow 2: Manual Installation (Boot Arch, Run Installer Manually)

If you need to customize disk layout or skip autorun:

```bash
# Boot Arch from USB
# Once in live environment:

mkdir -p /mnt/usb
mount /dev/sdX2 /mnt/usb
cd /mnt/usb/anthonyware

# Run installer manually (no autorun)
sudo ./install-anthonyware.sh
```

### Workflow 3: Disarm Autorun (Prevent Accidental Install)

To keep your USB but disable automatic installation:

```bash
# On running system with USB mounted:
rm -f /mnt/usb/anthonyware/USB_AUTORUN_ENABLED

# Now autorun will be disabled
# Installer only runs if you manually execute it
```

### Workflow 4: Update USB Repository

To pull latest Anthonyware from GitHub:

```bash
# On running system with USB mounted:
cd /mnt/usb/anthonyware

# Update repo to latest
bash scripts/usb-self-update.sh

# Verify integrity
bash scripts/usb-integrity-check.sh
```

### Workflow 5: Verify USB Integrity

Before installing, verify the repo is complete:

```bash
# On running system with USB mounted:
cd /mnt/usb/anthonyware

bash scripts/usb-integrity-check.sh

# Output:
# [ OK ] Git repository detected
# [ OK ] Working directory clean
# [ OK ] install-anthonyware.sh
# [ OK ] run-from-usb.sh
# ...
# ✓ Integrity check PASSED
```

## Safety Features

### Autorun Safety Lock

The autorun script will **not run automatically** unless:

1. ✅ The `USB_AUTORUN_ENABLED` marker file exists
2. ✅ The autorun service is installed in the live environment
3. ✅ The repo directory exists at the expected location

If any of these are missing → installer does **not** run automatically.

```bash
# Arm for autorun
touch anthonyware/USB_AUTORUN_ENABLED

# Disarm for manual-only
rm -f anthonyware/USB_AUTORUN_ENABLED
```

### Integrity Checking

Before installing, always verify:

```bash
cd /mnt/usb/anthonyware
bash scripts/usb-integrity-check.sh
```

This verifies:
- ✓ All required files present
- ✓ Git repository valid
- ✓ Working tree clean
- ✓ Scripts are executable
- ✓ Autorun status (armed/disarmed)

### Dry-Run Testing

Test the installer without modifying disks:

```bash
DRY_RUN=1 bash ./run-from-usb.sh
```

This will:
- Skip pacman operations
- Skip filesystem modifications
- Skip config deployments
- Show what WOULD happen

## Troubleshooting

### "USB_AUTORUN_ENABLED missing" - Autorun didn't start

**Solution**: Create the marker file

```bash
# On running system:
touch /mnt/usb/anthonyware/USB_AUTORUN_ENABLED

# Or before rebooting:
cd /mnt/usb/anthonyware
touch USB_AUTORUN_ENABLED
```

### "Repository not found" - Can't find USB

**Possible causes**:
1. USB not mounted at boot
2. Partition not readable
3. Anthonyware directory in wrong location

**Solution**:
```bash
# Boot into shell (don't use autorun)
# Manually mount and verify:
mkdir -p /mnt/usb
mount /dev/sdX2 /mnt/usb
ls -la /mnt/usb/anthonyware/

# If it exists, run manually:
cd /mnt/usb/anthonyware
sudo ./run-from-usb.sh
```

### "Integrity check failed" - Missing files

**Solution**:
```bash
# Verify files are on USB
cd /mnt/usb/anthonyware
bash scripts/usb-integrity-check.sh

# If files are missing, re-copy repo from source:
cp -r ~/anthonyware /mnt/usb/
```

### "Permission denied" on autorun binary

**Solution**:
```bash
# Make scripts executable
chmod +x usb/anthonyware-usb-autorun
chmod +x usb/install-autorun-into-live.sh
chmod +x run-from-usb.sh
chmod +x scripts/*.sh
```

## Advanced: Ventoy Configuration

Ventoy provides custom boot options through `ventoy.json`:

**Current config** (`usb/ventoy/ventoy.json`):
```json
{
  "image_list": [
    {
      "image": "/archlinux.iso",
      "alias": "Arch Linux - Anthonyware Installer"
    }
  ]
}
```

To customize:

1. Edit `usb/ventoy/ventoy.json` in your repo
2. Copy updated file to USB:
   ```bash
   cp usb/ventoy/ventoy.json /mnt/usb/ventoy/ventoy.json
   ```
3. Reboot to see changes
